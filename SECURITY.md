1. Signalement des vulnérabilités

Nous prenons la sécurité très au sérieux. Si vous découvrez une vulnérabilité dans ce projet, veuillez nous en informer de manière responsable afin que nous puissions la corriger rapidement.

Email pour signalement : recyclagemaria@gmail.com

Objet recommandé : [Sécurité] Description courte de la vulnérabilité

Merci de ne pas publier de vulnérabilité sur les réseaux publics avant que nous ayons eu le temps de la corriger.

2. Types de vulnérabilités à signaler

Vous pouvez signaler tout problème lié à :

Fuites de données personnelles ou sensibles

Failles XSS, CSRF, SQL Injection ou autres vulnérabilités web

Problèmes d’authentification ou d’autorisation

Toute autre faille de sécurité affectant le site

3. Processus de traitement

Vous nous envoyez le rapport de vulnérabilité par email.

Nous accusons réception sous 48 heures.

Nous analysons et confirmons le problème, puis travaillons à sa correction.

Une fois la vulnérabilité corrigée, nous pouvons informer le contributeur et, si approprié, publier un correctif dans le dépôt.

4. Récompenses / Reconnaissance

Bien que nous ne disposions pas actuellement d’un programme de récompense monétaire, les contributeurs qui signalent des vulnérabilités critiques seront mentionnés dans le fichier SECURITY.md ou dans les notes de version.

5. Bonnes pratiques de sécurité pour les contributeurs

Ne stockez jamais de mots de passe ou de clés API directement dans le dépôt.

Utilisez des variables d’environnement pour les informations sensibles.

Vérifiez vos dépendances régulièrement pour éviter les packages vulnérables.

Évitez d’inclure des données personnelles ou confidentielles dans les commits.
